## 要件定義

### 目的・背景
秘書またはCEOが業務上必要と判断した際、オーナーへの提案→承認→部署フォルダ自動生成までをワンフローで完結させる。「部署が必要になったら手動でフォルダを作る」手間をなくし、組織がAI主導で自律拡張できる仕組みを作る。

### 対象ユーザー
- 秘書（AI）・CEO（AI）: 提案者
- オーナー（渡邊氏）: 最終承認者

### 機能要件

| 優先度 | 機能 | 概要 |
|--------|------|------|
| 高 | 部署提案 | CEOが「この業務には専用部署が必要」と判断したら提案書を自動生成 |
| 高 | 承認フロー | `AskUserQuestion` でオーナーに承認/却下を確認 |
| 高 | フォルダ自動生成 | 承認後、`.company/[部署名]/` とCLAUDE.mdを自動作成 |
| 通常 | 部署テンプレート選択 | 既存テンプレート（営業/開発/リサーチ等）から選択または新規定義 |
| 通常 | 廃止フロー | 不要になった部署の廃止・アーカイブ提案 |
| 低 | 自律トリガー | 同種タスクがN回以上来たら自動で部署提案を起動 |

### 非機能要件
- 承認なしに部署は作成しない（必ずオーナー確認）
- 作成した部署はCLAUDE.mdに自動反映
- 部署乱立防止のため、既存部署との重複チェックを行う

## 詳細設計

### フロー設計

```
[トリガー]
同種タスクが3回以上 or CEOが手動判断
        ↓
[CEO: 提案書生成]
.company/ceo/decisions/YYYY-MM-DD-新部署提案-[名前].md を作成
        ↓
[AskUserQuestion]
「[部署名]を新設しますか？ 役割: [説明]」
YES / NO / カスタム
        ↓ YES
[自動生成]
1. .company/[部署名]/ フォルダ作成
2. CLAUDE.md（部署ルール）生成
3. _template.md 配置
4. .company/CLAUDE.md の組織構成セクションを更新
5. カレンダー + Discord に「新部署設立」ログ送信
```

### 部署提案書フォーマット (ceo/decisions/)

```markdown
---
type: department_proposal
status: pending | approved | rejected
proposed_by: CEO
proposed_at: YYYY-MM-DD
---
## 提案部署名
## 設立理由
（どんな業務が溜まっているか / なぜ既存部署では対応できないか）
## 想定する役割・業務範囲
## 既存部署との違い
## テンプレート種別
```

### 自動生成されるCLAUDE.md（部署用）

```markdown
# [部署名] 部署ルール

**設立日**: YYYY-MM-DD
**設立理由**: [CEOの判断理由]

## 役割
[部署の担当業務]

## 対応するタスクtype
[research / content_creation / idea / etc.]

## 保存先フォルダ
[3125XXX事業部/]

## 命名規則
YYYY-MM-DD-[タイトル].md
```

### 技術スタック（推奨）
- **実装場所**: `.company/CLAUDE.md` の指示として組み込み（コード不要）
- **フォルダ生成**: Bash `mkdir -p` + Write ツールで実現
- **CLAUDE.md更新**: Edit ツールで組織構成セクションに追記
- **通知**: 既存カレンダー + Discord curlコマンドを流用

## Claude Code MVP用プロンプト

```
# 秘書/CEO 自律部署作成権限システム MVP 実装依頼

## プロジェクト概要
`.company/CLAUDE.md` に「部署自律作成フロー」を追加し、
CEOが必要と判断したとき・または同種タスクが3回以上来たときに
自動で部署提案→オーナー承認→フォルダ生成までを完結させてください。

## 実装内容（コードではなくCLAUDE.mdへの追記）

### 1. トリガー条件をCLAUDE.mdに追記
以下の条件のいずれかに該当したらCEOが部署提案を起動する:
- 同種タスク（同じtype/業務内容）が3回以上 _pending/ に来た
- CEOが「この業務量には専用部署が必要」と判断した

### 2. 提案書自動生成
ceo/decisions/YYYY-MM-DD-新部署提案-[部署名].md を以下の構成で作成:
- 提案部署名
- 設立理由（どんなタスクが溜まっているか）
- 役割・業務範囲
- 既存部署との違い

### 3. AskUserQuestionで承認確認
選択肢: 「承認して作成」「却下」「部署名・役割を変更したい」

### 4. 承認後の自動処理
以下をBashとWriteツールで実行:
1. mkdir -p ".company/[部署名]/"
2. 部署CLAUDE.mdを生成（役割・対応タスクtype・保存先を記載）
3. _template.mdを配置
4. .company/CLAUDE.md の「組織構成」セクションに新部署を追記
5. カレンダー + Discord に完了通知を送信

## 完了条件
- CLAUDE.mdに上記フローが明確に記述されていること
- 提案書フォーマットが定義されていること
- 承認後のフォルダ生成手順がBashコマンドで記載されていること
- 既存部署との重複チェック手順が含まれていること
```
