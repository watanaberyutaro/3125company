# 💡「みんなの渡辺」AIエージェント アイデアブラッシュアップ提案書

**作成日:** 2026-03-15
**担当:** 企画部
**ステータス:** draft

---

## ① 元のアイデア整理

### コアコンセプト
**「みんなの渡辺」** — 渡邊氏のクローンAIエージェントが、人材営業の受付・マッチング・報告業務をLINE上で自律的に代行するシステム。

### 現状業務フロー（As-Is）
```
①担当者から連絡（LINE等）
   ↓
②スキルシートを受け取る
   ↓
③渡邊氏が案件を判断・マッチング
   ↓
④「この案件どうですか？」と返答
   ↓
⑤件数を集計・報告
⑥出退勤管理
```

### 求める機能
| 機能 | 概要 |
|------|------|
| LINE連携 | LINEでスキルシートを受け取り・提案を返信 |
| AIマッチング | スキルシートを解析して最適案件を提案 |
| 件数管理DB | 「今日XX件取りました」を記録・可視化 |
| 出退勤管理 | 出勤・退勤をLINEから記録 |
| 渡邊クローン | 渡邊氏の判断スタイル・口調を再現 |

---

## ② 課題・背景の分析

### 解決したい問題
1. **スケーラビリティの壁** — 人が増えると渡邊氏一人がボトルネックになる。AIが24時間対応することで件数を制限なく受け付けられる
2. **情報の属人化** — 判断基準が渡邊氏の頭の中にある。AIに学習させることで組織知化できる
3. **集計・管理の手間** — 件数報告・出退勤を手動で集計している。自動化で工数削減

### 市場背景
- 人材紹介・派遣業界はDX遅れが顕著。LINE活用は業界でも先行事例が少なく差別化になる
- AIエージェント市場が2026年に本格普及フェーズ（本日のニュースでも確認）
- 渡邊カンパニーがAI導入支援事業を展開しており、**自社で実証事例を作ることがセールス上の強みになる**

## ③ 展開案（3〜5つ）

---

### 案A：LINEボット型（最小構成）

**概要:** LINE公式アカウントにAIを接続。スキルシートをテキスト入力 or 添付で送ると、案件提案が自動返信される。

| 項目 | 内容 |
|------|------|
| 技術スタック | LINE Messaging API + Claude API |
| スキルシート解析 | テキスト抽出 → Claude でスキル分類 |
| 案件DB | Notionまたはスプレッドシート |
| 報告機能 | LINE送信 → Webhookでスプレッドシートに記録 |

**メリット**
- 最も低コスト・短期間で構築可能
- LINEは担当者も慣れているためオンボーディングが不要
- 渡邊カンパニーの既存技術（LINE Bot・Claude API）を活用できる

**懸念点**
- スキルシートがPDFの場合、テキスト抽出精度が課題
- 案件DBの整備が先決（案件情報がデータ化されていないと提案できない）

**収益モデル:** 社内利用 → 外部の人材会社向けSaaSとして横展開可能

---

### 案B：Webアプリ＋LINE連携型（標準構成）

**概要:** 管理画面（Web）＋LINE通知の2層構造。管理者は管理画面でダッシュボード確認、担当者はLINEで操作。

| 項目 | 内容 |
|------|------|
| フロントエンド | 管理ダッシュボード（Next.js等） |
| バックエンド | Node.js / FastAPI |
| DB | Supabase（PostgreSQL） |
| AI | Claude API（マッチング判定） |
| 通知 | LINE Messaging API |

**メリット**
- 件数・出退勤・案件情報を一元管理できるダッシュボードが実現
- 複数担当者・複数拠点に対応しやすい
- 将来的な機能拡張（AI学習・分析レポート）がしやすい

**懸念点**
- 開発工数が案Aより大きい（2〜3ヶ月想定）
- 運用・保守コストが発生する

**収益モデル:** 月額SaaS（1アカウント ¥30,000〜）として外販可能

---

### 案C：渡邊クローン特化型（差別化構成）

**概要:** 渡邊氏の過去の判断データ・口調を大量に学習させ、「渡邊クローン」としてのパーソナリティを前面に出したAIエージェント。「みんなの渡辺」ブランドで展開。

| 項目 | 内容 |
|------|------|
| クローン学習 | 過去の返信・判断事例をFine-tuning or Few-shot promptに投入 |
| 口調再現 | 渡邊氏のLINEメッセージ履歴から口調・ポリシーを抽出 |
| ブランド展開 | 「みんなの渡辺」として対外的にアピール |

**メリット**
- 「AI × 人格クローン」は現時点で競合ゼロの差別化ポイント
- マーケティング・PR価値が高い（メディア露出・SNSバズ狙い）
- 渡邊氏が不在でも質の高い対応ができる

**懸念点**
- 学習データの収集・整備に時間がかかる
- 判断ミスが起きた場合の責任設計が必要
- 個人情報（スキルシート）の取り扱いに注意

**収益モデル:** 同業他社への「AIクローン構築サービス」として展開（高単価・競合なし）

---

### 案D：社内ツール＋外販SaaS型（最大化構成）

**概要:** 案Bをベースに、社内で実証後に「人材営業向けAIエージェント」SaaSとして外販。

**メリット**
- 自社で使いながら改善できるため、リアルな実証事例を作れる
- 人材業界向けSaaSは単価が高く（月額¥50,000〜）、スケールしやすい
- 渡邊カンパニーのAI導入支援事業との相乗効果が最大

**懸念点**
- プロダクトマネジメントの工数が発生する
- 競合（Salesforce等大手）が本腰を入れてくると価格競争になるリスク

**収益モデル:** 月額SaaS ¥30,000〜50,000/社。10社で月300〜500万円

---

## ④ おすすめ案と理由

### 推奨: **案A → 案C の段階的展開**

```
Phase 1（1〜2ヶ月）: 案A（LINEボット最小構成）
  → まず社内で動くものを作り、業務フローを検証

Phase 2（3〜6ヶ月）: 案C要素を追加（渡邊クローン化）
  → 口調・判断スタイルを学習させ「みんなの渡辺」としてブランド確立

Phase 3（6ヶ月〜）: 案D（外販SaaS）
  → 社内実証を武器に人材業界へ横展開
```

**理由:**
1. 案Aから始めることでリスクを最小化しつつ、すぐに業務効果を得られる
2. 渡邊クローン化は差別化の核心。マーケ・PR効果も高く、AI導入支援事業の実績にもなる
3. 外販SaaSは渡邊カンパニーの収益多角化として最大のポテンシャルを持つ
