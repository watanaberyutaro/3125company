## 要件定義

### 目的・背景
オーナーが日常的に「ふと思ったこと」「アイデア」「判断の理由」を話しかけるだけで、AIが人格・思考パターンを継続的に学習し、将来的により「自分らしい」提案・意思決定サポートができる基盤を作る。

### 対象ユーザー
- 主: 渡邊カンパニー オーナー（渡邊氏）
- 将来: 秘書室アプリ外部展開時の一般ユーザー

### 機能要件

| 優先度 | 機能 | 概要 |
|--------|------|------|
| 高 | つぶやき入力 | `/company` 経由またはWebUIから短文・長文を入力 |
| 高 | 自動分類保存 | 入力内容を「アイデア」「判断」「価値観」「感情」に自動分類し `secretary/inbox/` に保存 |
| 高 | 人格サマリー生成 | 月次でCEOが蓄積データから人格サマリーを作成・更新 |
| 通常 | タグ付け | 業務領域（営業/開発/経営など）で自動タグ付け |
| 通常 | 検索・参照 | 過去のつぶやきを全文検索できる |
| 低 | 音声入力連携 | iPhoneボイスメモ→テキスト変換→自動送信 |

### 非機能要件
- 入力から保存まで3秒以内
- Obsidian Vaultと同期（Markdown形式で保存）
- プライバシー: データはローカル/iCloud内のみ（外部送信なし）


**② 人格サマリー画面（管理者向け）**
```
┌─────────────────────────────┐
│ 🧠 人格サマリー（最終更新: XX/XX）│
│ ─────────────────────────── │
│ 判断軸: スピード > 完璧主義     │
│ 価値観: 自律・実用主義・成長    │
│ 関心領域: AI, 人材, スマホ     │
│ ─────────────────────────── │
│ 最近のつぶやき（5件）           │
└─────────────────────────────┘
```

### データモデル

**つぶやきレコード** (`secretary/inbox/YYYY-MM-DD-HH-mm-つぶやき.md`):
```markdown
---
type: thought
category: idea | decision | value | emotion
tags: [営業, AI, ...]
created: YYYY-MM-DDTHH:mm:ss+09:00
---
[つぶやき本文]
```

**人格サマリー** (`.company/secretary/personality-summary.md`):
```markdown
---
updated: YYYY-MM-DD
version: N
---
## 判断軸
## 価値観
## 行動パターン
## 関心領域
## 最近の傾向
```

### API設計

| エンドポイント | メソッド | 概要 |
|--------------|---------|------|
| `/api/thought` | POST | つぶやきを受け取り分類・保存 |
| `/api/thoughts` | GET | つぶやき一覧取得（検索・フィルタ対応） |
| `/api/personality` | GET | 人格サマリー取得 |

### 技術スタック（推奨）
- **フロントエンド**: 既存WebUI（秘書室アプリ）に入力UIを追加
- **バックエンド**: 既存Vercel Functions（Node.js）に `/api/thought` を追加
- **AI分類**: Claude API（Haiku）でカテゴリ・タグを自動付与
- **ストレージ**: iCloud経由でObsidian Vault（Markdownファイル）に保存
- **検索**: Obsidianの全文検索機能を活用

## Claude Code MVP用プロンプト

```
# 思考発話・人格理解システム MVP 実装依頼

## プロジェクト概要
ユーザーが日常的に「ふと思ったこと」を入力すると、AIが自動分類して
Obsidian Vault（Markdown）に保存する「パーソナル思考ログシステム」を作成してください。

## 使用技術
- Next.js 14（App Router）
- Vercel Functions（APIルート）
- Claude API（claude-haiku-4-5: カテゴリ分類・タグ付け用）
- ファイルシステム（iCloud/ローカルのMarkdownファイルへの書き込み）

## 実装してほしい機能（優先度順）

### 必須（MVP）
1. **入力UI**: テキストエリア + 送信ボタン（シンプルなチャット風）
2. **自動分類API** (`POST /api/thought`):
   - Claude Haikuで入力テキストを以下に分類:
     - category: "idea" | "decision" | "value" | "emotion" | "memo"
     - tags: 関連業務領域の配列（例: ["AI", "営業", "経営"]）
   - Markdownファイルとして保存:
     - パス: `[VAULT_PATH]/secretary/inbox/YYYY-MM-DD-HH-mm-[category].md`
     - frontmatter: type, category, tags, created
3. **一覧表示**: 保存されたつぶやきを新着順で表示

### 任意（時間があれば）
4. **キーワード検索**: タイトル・本文をローカル検索
5. **人格サマリー生成**: 蓄積されたつぶやきからClaude Opusが月次サマリーを生成
```
