# 💡 TARS（インターステラー）型ミニロボット

**作成日**: 2026-03-16
**担当**: 3125アイデア保管事業部（アイゼン）

---

## ① 元のアイデア整理

**コアコンセプト**
映画「インターステラー」に登場するAIロボット「TARS」をモチーフにした、物理ロボット型AIデバイスの開発。TARSの特徴（可変形状・ユーモア設定・誠実な答え・高い信頼性）を持つ小型AIアシスタントロボット。

---

## ② 課題・背景

- AIは現在スクリーンの中にある。物理的な存在感・キャラクター性を持つデバイスへの需要がある
- スマートスピーカー（Alexa/Google Home）は機能的だが個性・愛着が薄い
- TARSはSF史上最も「実用的かつ個性的」なAIキャラとして人気が高い
- 「ロボット×AI秘書」という組み合わせは、渡邊カンパニーのAI事業との親和性が高い
- ハードウェア開発は難易度が高いが、ソフトウェア（キャラクターAI）は事業強みを活かせる

---

## ③ 展開案

### 案A: ソフトウェアのみ（最小リスク・推奨）
Raspberry Pi + スピーカー + 簡易ケースで「TARS風外観のAIデバイス」を構築。
ソフトウェアはClaude API + フリーレン秘書システムを転用。
- **メリット**: 開発コスト低（数万円〜）・すぐ試作可能
- **懸念点**: 外観の完成度に限界がある
- **収益モデル**: 個人向けキット販売・DIYブログ→集客

### 案B: 3Dプリント筐体 + Raspberry Pi
TARS形状を3Dプリントで作成。モーター制御で変形ギミック追加。内部にRaspberry Pi + マイク + スピーカー。
- **メリット**: 外観再現度が高い・話題性あり
- **懸念点**: 3D設計・モーター制御の技術ハードル。製造コスト上昇
- **収益モデル**: 限定販売・Makersマーケット

### 案C: AIキャラクタープラットフォームとしての展開
TARSロボットをベースに、「任意のキャラクターAIを搭載できる物理デバイス」として事業化。
フリーレン・ヒンメル・アイゼン等のキャラを選べるサブスク型。
- **メリット**: スケーラブル・サブスク収益
- **懸念点**: 著作権リスク（キャラクター利用）・ハードウェア量産ハードル
- **収益モデル**: デバイス販売（初期） + キャラサブスク（月額）

### 案D: エンタープライズ向け「受付AIロボット」
TARSデザインの卓上型受付AIとして企業・クリニック・店舗に展開。
来客対応・FAQ応答・導線案内を担当。
- **メリット**: BtoB・高単価
- **懸念点**: 競合（SoftBank Pepper等）との差別化が必要

---

## ④ おすすめ案

**案A（Raspberry Pi + ソフトウェア）でプロトタイプ制作 → SNSで公開 → 案Bへ発展**

まずRaspberry Pi + Claude APIで「TARSっぽい会話ができる箱」を作り、X/YouTube/TikTokに投稿して反響を見る。
バズれば案Bへ発展。その過程で案Cのキャラクタープラットフォーム構想のニーズも検証できる。
